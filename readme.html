Also note that "football" as used here refers to what most of the world knows as "football". In other words, "soccer" for those of us in the USA.

About the Program

As the name suggests, this program is a football league or cup simulator. Well, maybe "simulator" isn't quite the right word, it's not exactly all that in-depth, but at any rate it will simulate an entire league season's worth of matches in leagues of any size, and can also generate results for cup-style competitions as well.

The name "NSFootySim", adopted in its current form as of version 0.8.3a, is a reference to the original purpose of the program, generating results for the domestic leagues of fictional nations in the NationStates online game. I have since realized that it can have applications outside of that game, though, but the "NS" has been retained just to make sure I don't run afoul of someone else's trademark.

Also as of version 0.8.3a, the program is released under the GNU General Public License. The requisite source code is included with the program.

I. Installation

  1. Simply unzip all the files in this archive to the directory of your choice. That's it!

II. How to Use This Program

  1. Generating results from a custom league
    1. Create the league definition XML file. The file sample_input.xml is provided as an example, and it includes some comments which may be useful.
    2. Start the program by running NSFootySim.jar.
    3. Press the "League" button.
    4. A dialog box will appear. Select the league definition file and press OK.
    5. A second dialog box will appear. Select the directory and type in the name of the file to which you want to save the match results. Press OK.
    6. A third dialog box will appear. Select the directory and type in the name of the file to which you want to save the league table. Press OK.
    7. In addition, the file resume.xml will be output to your NSFootySim directory. If you have generated only part of a season, you should use this file as the input file when you continue.
  2. Generating a single-elimination tournament
    1. Create the cup definition XML file. The file sample_cup.xml is provided as an example, and it includes some comments which may be useful.
    2. Start the program by running NSFootySim.jar.
    3. Press the "Cup" button.
    4. A dialog box will appear. Select the cup definition file and press OK.
    5. A second dialog box will appear. Select the directory and type in the name of the file to which you want to save the match results. Press OK.

III. Customizing the Formulae

To customize the formulae used for generating match results, open the included file formula.cfg and edit the numbers (ONLY the numbers, nothing else!) on each line.

Each variable is used in the generation process as follows (variables in blue are those found in formula.cfg):

Number of attacks (sequential rankings, stronger team):
(baseAttacksSequentialSuperior + ((strongerTeamRank - weakerTeamRank) * attackCoeffSequentialSuperior)) * ((homeStyleModifier + awayStyleModifier + styleOffset) ^ styleScalar * styleCoeff) (rounded down)

Number of attacks (sequential rankings, weaker team):
(baseAttacksSequentialInferior + ((strongerTeamRank - weakerTeamRank) * attackCoeffSequentialInferior)) * ((homeStyleModifier + awayStyleModifier + styleOffset) ^ styleScalar * styleCoeff) (rounded down)
(usually attackCoeffSequentialInferior will be either zero or a negative number)

Number of attacks (point-based rankings, stronger team):
(baseAttacksPointBasedSuperior + ((strongerTeamRank - weakerTeamRank) / maxRankPoints) * attackCoeffPointBasedSuperior) * ((homeStyleModifier + awayStyleModifier + styleOffset) ^ styleScalar * styleCoeff) (rounded down)

Number of attacks (point-based rankings, weaker team):
(baseAttacksPointBasedInferior + ((strongerTeamRank - weakerTeamRank) / maxRankPoints) * attackCoeffPointBasedInferior) * ((homeStyleModifier + awayStyleModifier + styleOffset) ^ styleScalar * styleCoeff) (rounded down)
(usually attackCoeffPointBasedInferior will be either zero or a negative number)

Attack success threshold (sequential rankings, home team):
baseAttackSuccessThresholdSequential - homeBonusSequential + (homeRank - awayRank) * attackSuccessCoeffSequential
(use basePenAttackSuccessThresholdSequential instead of baseAttackSuccessThresholdSequential for penalty shootouts)

Attack success threshold (sequential rankings, away team):
baseAttackSuccessThresholdSequential + homeBonusSequential + (awayRank - homeRank) * attackSuccessCoeffSequential
(use basePenAttackSuccessThresholdSequential instead of baseAttackSuccessThresholdSequential for penalty shootouts)

Attack success threshold (sequential rankings on neutral ground, "home" team):
baseAttackSuccessThresholdSequential + (homeRank - awayRank) * attackSuccessCoeffSequential
(use basePenAttackSuccessThresholdSequential instead of baseAttackSuccessThresholdSequential for penalty shootouts)

Attack success threshold (sequential rankings on neutral ground, "away" team):
baseAttackSuccessThresholdSequential + (awayRank - homeRank) * attackSuccessCoeffSequential
(use basePenAttackSuccessThresholdSequential instead of baseAttackSuccessThresholdSequential for penalty shootouts)

Automatic attack success threshold (sequential rankings):
autoSuccessThresholdSequential

Automatic attack failure threshold (sequential rankings):
autoFailureThresholdSequential

Attack success inequality (point-based rankings, home team):
attack * (baseAttackCoeffPointBased + homeBonusPointBased) - rankDiffModifierCoeffPointBased * (((awayRank / maxRankPoints) * rankCoeffPointBased) ^ rankScalarPointBased) - ((homeRank / maxRankPoints) * rankCoeffPointBased) ^ rankScalarPointBased) > baseAttackSuccessThresholdPointBased
(use basePenAttackSuccessThresholdPointBased instead of baseAttackSuccessThresholdPointBased for penalty shootouts)

Attack success inequality (point-based rankings, away team):
attack * (baseAttackCoeffPointBased - homeBonusPointBased) - rankDiffModifierCoeffPointBased * (((homeRank / maxRankPoints) * rankCoeffPointBased) ^ rankScalarPointBased) - ((awayRank / maxRankPoints) * rankCoeffPointBased) ^ rankScalarPointBased) > baseAttackSuccessThresholdPointBased
(use basePenAttackSuccessThresholdPointBased instead of baseAttackSuccessThresholdPointBased for penalty shootouts)

Attack success inequality (point-based rankings on neutral ground, "home" team):
attack * baseAttackCoeffPointBased - rankDiffModifierCoeffPointBased * (((awayRank / maxRankPoints) * rankCoeffPointBased) ^ rankScalarPointBased) - ((homeRank / maxRankPoints) * rankCoeffPointBased) ^ rankScalarPointBased) > baseAttackSuccessThresholdPointBased
(use basePenAttackSuccessThresholdPointBased instead of baseAttackSuccessThresholdPointBased for penalty shootouts)

Attack success inequality (point-based rankings on neutral ground, "away" team):
attack * baseAttackCoeffPointBased - rankDiffModifierCoeffPointBased * (((homeRank / maxRankPoints) * rankCoeffPointBased) ^ rankScalarPointBased) - ((awayRank / maxRankPoints) * rankCoeffPointBased) ^ rankScalarPointBased) > baseAttackSuccessThresholdPointBased
(use basePenAttackSuccessThresholdPointBased instead of baseAttackSuccessThresholdPointBased for penalty shootouts)

The default settings, in case you should need to return to them, are:

  • baseAttacksSequentialSuperior 10
  • baseAttacksSequentialInferior 10
  • baseAttacksPointBasedSuperior 10
  • baseAttacksPointBasedInferior 10
  • attackCoeffSequentialSuperior 0.05
  • attackCoeffSequentialInferior 0
  • attackCoeffPointBasedSuperior 10
  • attackCoeffPointBasedInferior 0
  • baseAttackSuccessThresholdSequential 0.8695
  • basePenAttackSuccessThresholdSequential 0.4
  • homeBonusSequential 0.0165
  • attackSuccessCoeffSequential 0.003
  • autoSuccessThresholdSequential 0.95
  • autoFailureThresholdSequential 0.75
  • baseAttackCoeffPointBased 667.5
  • homeBonusPointBased 7.5
  • baseAttackSuccessThresholdPointBased 580
  • basePenAttackSuccessThresholdPointBased 400
  • rankDiffModifierCoeffPointBased 12
  • rankScalarPointBased 0.5
  • rankCoeffPointBased 31.5
  • styleOffset 20
  • styleScalar 1.5
  • styleCoeff 0.01118

v0.5.2 (2004-08-27)

  • The Cup is back! You can now use any standard league definition file to generate a single-elimination tournament. Right now, the first round team matchups are randomized, though this is expected to be made optional later. Currently, only the "Neutral" league option works, but home/away support is expected to be added soon. Also note that the "Allow Draws" setting here is irrelevant; draws cannot be allowed. Please also note that cup tournament generation will only work if the number of teams in the league definition file is a power of two. If it isn't, nothing will happen.

v0.4.3 (2004-06-09)

  • At long last, the BFA Scorinating Utility can handle odd numbers of teams in a league! You can now generate a league with any number of teams from 2 to (theoretically, untested) 32,767.
  • The program now uses progress bars to keep the user informed of the progress of a generation job.
  • You can now determine the number of points awarded to a team for a win, a draw, and a loss. This allows for modifications such as the Rejistanian point system (4/2/1 instead of 3/1/0). Wins, draws, and losses can all be set within the range 0-100, inclusive. Be sure not to go too high, though, because final team point totals of 1000 or higher will make the league table look weird.
  • You can now choose whether to simulate each league with each team playing each other team once home and away, or with each team playing each other team once on neutral ground. The "neutral ground" has not truly been implemented yet, though; one team still has home team advantage. Neutrality will be implemented in the next update.
  • You can now choose whether or not to allow draws in the league (particularly useful when simulating single cup matches).

v0.4.2 (2004-06-07)

  • You may now choose from a dialog box which file to import a league definition from when working with a custom league. This also effectively brings back the ability to generate a single BFA division without having to generate the other divisions.

v0.4.1 (early June 2004)

  • The ability to simulate a custom league is now available. Simply select the "Custom League" radio button and the program will generate a full league's worth of matches for the teams in the file "custom_teams.txt" in the Teams folder.
  • The "Show Results" and "Clear" buttons have been removed. "Show Results" is no longer feasible due to the very high possibility of the custom league containing a very large number of teams, and the "Clear" button is no longer necessary because the program now automatically clears all variables before generating a new league or division.
